4. I have another bug, it shows that the default profile ~/.claude is not logged in. I have made a research and found the following:
On macOS, Claude Code stores OAuth credentials in the Keychain, and the service name depends on whether
  CLAUDE_CONFIG_DIR is set:
  - Unset (default): it uses the plain service name Claude Code-credentials — that's where your Max login lives, so claude auth status
  shows you logged in.
  - Explicitly set — even to ~/.claude, the same directory as the default: it uses a suffixed service name, Claude Code-credentials-<hash
  of the config dir path>. 
  In other words, "no CLAUDE_CONFIG_DIR" and "CLAUDE_CONFIG_DIR=~/.claude" are two different credential namespaces even though they resolve
  to the same directory on disk.
